# Пошаговый бой + рандом раздачи + козыри-одноразовые

## Цель
Переделать дуэль из одновременного вскрытия в **пошаговую**: противник ходит первым,
игрок отвечает. Больше рандома (разная раздача каждый бой). Колода врага сильнее с
ростом уровня. Козыри доступны сразу (рука 4 + козыри), каждый козырь — 1 раз за бой,
в следующем бою возвращается.

## Ход боя
Раунд: противник вскрывает карту → игрок отвечает **одной** картой (из руки или
козырь) → `DuelEngine.resolveRound` (без изменений: урон = разница эффективных сил,
×1.5 doubleStrike, shield→0) → урон по замку проигравшего. Повтор, пока чей-то замок
не падёт (`outcome` без изменений). Игрок всегда отвечает картой (пропуска нет).

## Карты игрока: рука 4 + козыри
- **Рука**: 4 обычные карты, добираемые из перетасованной колоды owned не-козырей;
  после хода картой из руки — добор до 4 (колода recycle при опустошении).
- **Козыри**: все owned-козыри — отдельный набор `availableTrumps`, каждый играется
  один раз за бой. Сыграть козырь = ответ на ход; он удаляется из `availableTrumps`
  (на бой). Руку козырь не пополняет. В следующем бою колода/козыри собираются заново
  из owned, поэтому козыри возвращаются.

## Колода врага — взвешенная по силе по уровню
Собирается из общего пула карт с уклоном в силу, растущим с уровнем узла
(`MapNode.level`: тренировка=0, п1=1, п2=2, босс=3): вес карты ∝ `power^(1+level)`,
так что выше уровень → заметно больше сильных карт. Босс дополнительно включает свой
сильный набор + козырь (`rewardTrumpId`). Колода тасуется свежим рандомом → разные
карты каждый бой. Размер колоды — `kDeckSize` (как сейчас).

## Рандом
Каждый бой использует свежий `Random()` (без seed) → раздача всегда разная.
`buildSession`/`DuelSession` принимают `Random` как параметр; продакшн (`duel_screen`)
передаёт `Random()`, тесты — `Random(seed)`.

## API сессии (`duel_session.dart`)
- `start()` — тасует обе колоды, раздаёт 4 в руку игрока и 4 врагу, ставит HP, затем
  противник выкладывает первую карту → `pendingOpponentCard`.
- `GameCard get pendingOpponentCard` — карта, на которую отвечает игрок (null только
  если у врага кончились карты, чего не происходит из-за recycle).
- `List<GameCard> get availableTrumps` — неиспользованные козыри игрока.
- `List<GameCard> get playerHand` — 4 обычные карты.
- `RoundResult respond(GameCard card)` — `card` это карта руки ИЛИ козырь:
  - резолв `pendingOpponentCard` против `card`; урон по замку;
  - если `card` из руки → удалить и добрать до 4; если козырь → удалить из
    `availableTrumps` (не добирать);
  - противник выкладывает следующую карту в `pendingOpponentCard`;
  - вернуть `RoundResult`.
- `DuelOutcome get outcome` — без изменений.

`AiController.chooseCard` (выбор сильнейшей карты) переиспользуется для хода врага —
теперь врага зовём первым каждый раунд.

## UI (`duel_screen.dart`)
- Карта `pendingOpponentCard` лежит на столе лицом вверх (в зоне столкновения).
- Игрок тапает карту руки **или** козырь → анимация столкновения (текущая) →
  результат/пояснение → новая карта противника.
- Низ: ряд 4 карт руки + ряд козырей; использованный/потраченный козырь гаснет
  (dimmed, не тапается).

## Тесты
- `resolveRound` без изменений — его тесты остаются.
- Новые тесты `DuelSession`: противник ходит первым (`pendingOpponentCard` ≠ null
  после `start`); `respond` считает урон и атрибутирует замку; ход картой руки →
  добор до 4; ход козырём → козырь исчезает из `availableTrumps`, рука не меняется;
  разные seed → разные раздачи; колода врага сильнее с уровнем (средняя сила растёт).
- Обновить существующие тесты сессии/сетапа под новый API (`respond` вместо
  `playPlayerCard`, `pendingOpponentCard`, `availableTrumps`).
