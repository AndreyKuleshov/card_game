# Объединённая «Карта мира» + королевство

## Цель
Слить два экрана (`WorldMapScreen` со списком боёв и `KingdomScreen` со сценой-деревней
и прокачкой) в один игровой экран. Королевство со стеной — в центре; тренировка —
внутри стены; противники и босс — снаружи.

## Экраны и навигация
- `WorldMapScreen` — единственный игровой экран (home): сцена-королевство (замок,
  стена-периметр, постройки) + узлы боёв поверх неё.
- `KingdomScreen` удаляется; его виджеты сцены (`_VillageScene`, `_MeadowPainter`,
  `_WallFramePainter`, `_PositionedBuilding`, `_UpgradePanel`) переезжают в общий экран
  (вынести в переиспользуемый виджет, чтобы файл экрана не разрастался).
- Шапка: чип кристаллов + кнопка «Коллекция». Кнопка «Королевство» убирается.
- Тап по постройке → панель прокачки (как сейчас). Тап по узлу боя → `DuelScreen(node)`.

## Узлы боёв
- **Тренировка** — маркер внутри стены (у замка). Всегда доступна и повторяема.
- **Текущий противник** — один маркер снаружи стены (узел с `index == frontier`,
  если это противник). Побеждён → исчезает, появляется следующий.
- **Босс** — маркер снаружи, виден всегда; заблокирован (🔒) пока `frontier < bossIndex`;
  доступен при `frontier >= bossIndex`.

## Модель прогресса
- Переиспользуем `SaveState.unlockedNodeIndex` как «фронтир».
- Узлы `kSliceNodes`: `0`=Тренировка, `1..N-2`=противники, `N-1`=босс.
- Противник доступен ⇔ `node.index == frontier` и `node.index < bossIndex`.
- Босс доступен ⇔ `frontier >= bossIndex`.
- Тренировка доступна всегда (frontier не влияет).
- Победа над текущим противником → `frontier++` (через `unlockNextNode`). Тренировка и
  повторные бои фронтир не двигают.

## Награды (`computeDuelReward`)
Базовая сумма зависит от типа узла, сверху всегда прибавляется бонус шахты
(`kingdom.mineCrystalsPerWin`):
- Тренировка (`index == 0`): **5** + шахта. Без шанса козыря.
- Противник (`1 <= index < bossIndex`): **10 + 5·index** (Противник 1→15, 2→20) +
  шахта + 30% шанс выпадения козыря (`trump_frost_granny`).
- Босс: **30** + шахта + гарантированный козырь `rewardTrumpId` (`trump_pumpkin_king`).
- `unlockNext` = `won && node.index == frontier && node.index < bossIndex`
  (тренировка не двигает фронтир, т.к. её index 0 < frontier после первого противника;
  на старте frontier=0 и тренировка == frontier, поэтому добавляем явное условие
  «узел не тренировка» либо стартуем frontier=1 — см. ниже).

### Стартовое значение фронтира
Чтобы тренировка (index 0) не считалась «текущим противником», начинаем
`unlockedNodeIndex = 1` (первый противник). `SaveState.initial()` и миграция: если
загруженное значение `< 1`, поднять до 1.

## Раскладка (черновая, шлифуется вживую)
- Тренировка — внутри периметра, у замка.
- Противник — снаружи стены, верх-лево (поле/небо за стеной).
- Босс — снаружи стены, верх-право.

## Проверки
- `dart analyze lib/` чисто; `flutter test` зелёный (обновить/добавить тесты
  `computeDuelReward` под новые суммы и прогресс).
- Живой прогон: тренировка повторяема (+5), победа над противником открывает
  следующего и убирает побеждённого, босс разблокируется после всех противников.
